Where does React come from?

  • Facebook production: battle-tested, scalable, supported
  • User Interface: focused, limited
  • a solution to modern UI issues:
    • synchronizing multiple DOM elements
    • event storms
    • maintainability / evolutivity
“React is a great solution to a real problem backed by a rich and talented company.”
## React is not - a framework, a stack, a paradigm, etc. - a silver bullet - a fad - hard (especially with Meteor :) - easy

## a community/ecosystem - off-the-shelf components: [Material UI](http://www.material-ui.com/),... - tooling: [React Dev Tools](https://github.com/facebook/react-devtools), IDE, composers, [Radium](http://stack.formidable.com/radium/),... - other targets: [React-Native](https://facebook.github.io/react-native/) - ...
## some best practices - everything is a component - [Smart and Dumb](https://medium.com/@dan_abramov/smart-and-dumb-components-7ca2f9a7c7d0#.qlto1zdgw) - [lifecycle methods](http://facebook.github.io/react/docs/component-specs.html#lifecycle-methods) - performance: shouldComponentUpdate, [profiler](http://facebook.github.io/react/docs/perf.html) - beware [mixing jQuery and React](https://discuss.reactjs.org/t/jquery-with-react/683/4)
## Why React? - design your UI code in a great way - keep performance and maintainability - benefit from the ecosystem
